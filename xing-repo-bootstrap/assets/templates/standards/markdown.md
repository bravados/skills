# Standard: Markdown Writing

## Principles
- Prefer clarity over cleverness.
- Keep docs easy to scan.

## Rules
- Use one top-level heading (`#`) per file.
- Keep headings short; avoid nested depth beyond 3 levels.
- Use lists for steps; keep items concise.
- Use fenced code blocks with an info string.
- Avoid raw URLs in prose; use reference-style links when needed.

## Examples
- Good:
  - "## Setup"
  - "- Run `make test`"
- Avoid:
  - "### ###"
  - Walls of text without headings

## Validation
- A reader should find key info within 30 seconds.
